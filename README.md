# 🏰 PixelDungeon Dash: D&D Edition

Высокодетализированный бесконечный раннер в стиле темного фэнтези, созданный специально для Telegram WebApp. Исследуйте опасные глубины подземелья, сражайтесь с монстрами и соревнуйтесь в глобальном Зале Славы!

---

## 🏗️ Архитектура системы

Игра построена на базе **Next.js 15** и использует гибридную модель обработки данных:

1.  **Game Engine (Client-side)**: Кастомный игровой движок на HTML5 Canvas с частотой 60 FPS. Вся физика, анимация монстров и расчет коллизий происходят на стороне клиента для обеспечения максимальной отзывчивости.
2.  **State Management**: Состояние персонажа (HP, логи боя, выбранный класс) управляется через **React Context API**, что позволяет синхронизировать UI-элементы (сердечки, логи) с игровым процессом.
3.  **Backend-as-a-Service (Firebase)**: 
    *   **Firestore**: Хранение глобальных рекордов в реальном времени.
    *   **Auth**: Анонимная авторизация для идентификации игроков без ввода паролей.
4.  **Telegram Integration**: Бесшовная работа внутри Telegram WebApp с автоматическим получением данных профиля пользователя для таблицы лидеров.

## 📂 Структура проекта

```text
src/
├── ai/                 # Интеграция Genkit для будущих AI-функций (диалоги, генерация уровней)
├── app/                # Next.js App Router (страницы, макеты, API роуты)
│   ├── api/            # Серверные эндпоинты (валидация и сохранение данных)
│   └── globals.css     # Глобальные стили, переменные темы Shadcn и анимации (тряска экрана)
├── components/         # React-компоненты
│   ├── ui/             # Библиотека компонентов Shadcn (кнопки, диалоги, карточки)
│   ├── GameCanvas.tsx  # Ядро игры: рендеринг и логика игрового цикла
│   └── Leaderboard.tsx # Компонент глобальной таблицы рекордов
├── context/            # Контексты приложения (D&D механики и состояние игрока)
├── firebase/           # Конфигурация и хелперы для работы с Firebase SDK
│   ├── firestore/      # Кастомные хуки для real-time подписок (useCollection, useDoc)
│   └── non-blocking/   # Оптимизированные функции записи данных без ожидания (Fire and Forget)
├── hooks/              # Кастомные React-хуки (игровой цикл, определение мобильных устройств)
├── lib/                # Чистая логика и утилиты
│   ├── dnd-logic.ts    # Правила D&D: расчет AC, броски d20, статы классов
│   ├── game-math.ts    # Математика игры: коллизии AABB, расчет прогрессии скорости
│   └── asset-manifest.ts# Конфигурация визуальных параметров и хитбоксов монстров
└── types/              # TypeScript интерфейсы и определения типов
```

## 🛣️ Основные маршруты

-   **`/` (Home)**: Основной игровой экран. Включает в себя холст игры, выбор персонажа и таблицу лидеров.
-   **`/api/game/score`**: API-эндпоинт для сохранения результатов (используется как альтернатива прямой записи в Firestore для дополнительной валидации на сервере).

---

## 🛡️ Игровые механики D&D

-   **Класс Брони (AC)**: При столкновении с монстром происходит автоматическая проверка. Бросается виртуальный кубик d20. Если результат + бонусы выше AC монстра, вы блокируете удар.
-   **Уникальные классы**:
    -   **Fighter**: Танк с высоким AC (15) и запасом здоровья.
    -   - **Rogue**: Мастер маневров с пассивным двойным прыжком.
    -   - **Wizard**: Бонус +25% к опыту (дистанции).
    -   - **Bard**: Пассивная регенерация здоровья со временем.

---

## 👾 Бестиарий (Детализированные монстры)

-   **Древний Дракон**: Огромный противник с анимированными перепончатыми крыльями и многослойным извергающимся пламенем.
-   **Мимик**: Жуткий сундук, который клацает зубами и следит за вами множеством глаз.
-   **Огр**: Массивный враг с огромной шипованной дубиной.
-   **Бехолдер, Слизь, Призраки и Летучие мыши**: У каждого свой паттерн движения и уникальные хитбоксы.

## 🚀 Технологический стек

-   **Frontend**: Next.js 15, React 19, TypeScript.
-   **Styling**: Tailwind CSS, Lucide Icons.
-   **Database**: Firebase Firestore (Leaderboard).
-   **Auth**: Firebase Anonymous Auth.
-   **AI**: Genkit (Google Gemini integration).
-   **UI Components**: Radix UI (через Shadcn).
